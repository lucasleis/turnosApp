# turnosApp — Contexto del proyecto

## Producto
App web multi-tenant de reserva de turnos para barberías, escalable a otros rubros.

## Roles
- Cliente: reserva, cancela y reprograma turnos
- Barbero: ve su agenda y gestiona disponibilidad
- Admin: gestión completa del negocio

## Forma de trabajo
- Avanzar módulo por módulo, esperando aprobación antes de continuar
- No asumir decisiones importantes sin consultar al product owner
- Código modular y escalable desde el inicio
- Mobile-first y responsive
- No agregar dependencias innecesarias

## Orden de módulos
1. Stack y estructura del proyecto
2. Base de datos y modelos
3. Autenticación y roles
4. Flujo del cliente
5. Panel de administración
6. Notificaciones
